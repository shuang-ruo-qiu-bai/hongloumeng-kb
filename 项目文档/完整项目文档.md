## 一、项目总览

**红楼梦研究**（又名"红楼梦寻"）是一个中文《红楼梦》研究综合网站，包含原著阅读、红学历史、学者传记、版本研究、风俗考据、诗词赏析、电子书库、AI 问答、语义/关键词搜索等功能。

### 技术栈

| 层 | 技术 |
|----|------|
| 后端 | Python 3 + Flask + gunicorn |
| 前端 | Jinja2 模板 + 原生 CSS/JS（无框架） |
| 认证 | Flask-Login（无 CSRF） |
| AI | LLM API（DeepSeek/OpenAI 等，可配置） |
| 搜索 | Chroma 向量库 + 句嵌入 ONNX 模型（768维） |
| 数据库 | SQLite（用户/会话/消息） |
| 部署 | systemd，单 worker + 4 threads（gthread） |

### 代码位置

- **远程服务器**：`root@114.67.247.66:/opt/hongloumeng/`
- **本地副本**：`server_files/` 目录
- **临时脚本**：`server_scripts/` 目录

### 3.3 缺失的路由

**`/books/<path>`** — 不存在！`library.html` 中每个图书卡片链接到 `/books/{{ b.path }}`，但没有 Flask 路由处理这个路径，点击下载返回 404。需要添加：

```python
@app.route("/books/<path:filename>")
def download_book(filename):
    books_dir = "/opt/hongloumeng/books/红楼梦"
    return send_from_directory(books_dir, filename, as_attachment=True)
```

### 4.1 样式系统（style.css）

布局：

- 首页 `.main-content`：`display: flex; flex-direction: column; min-height: calc(100vh - 52px)`，`quote-bar` 用 `margin-top: auto` 推到底部

### 4.2 首页边栏（最近的改动）

`index.html` 独占，其他页面不可见：

- `<aside class="index-sidebar">`：`position: fixed; top: 52px; left: 0; width: 250px; z-index: 90`
- `.sidebar-handle`：固定定位的拖拽手柄，`cursor: col-resize`
- 宽度从 `localStorage('hlm_sidebar_width')` 恢复
- `.main` 的 `padding-left` 同步补偿

**实现文件**：
- `app/templates/index.html` — 边栏 HTML + head 样式覆盖
- `app/static/style.css` — 边栏所有样式规则
- `app/static/script.js` — `loadSessions()` / `newChatSession()` / `switchSession()` / `toggleSidebar()` / 拖拽 IIFE

**问题**：
- `.main` 的 `max-width: 1200px + padding-left: 268px` 导致窄屏溢出（已加 `body { overflow-x: hidden }` 缓解）
- AI 问答按钮疑似不响应（可能 z-index 或 CSS 偏移问题）
- 底部名言区（`#quote-bar`）可能不可见（需要检查 `min-height` 和 flex 布局）

### 4.4 原著章节 API（api_canon_chapter）

正文/注释分离逻辑：
- 行首正则匹配 `[1]` 或 `〔一〕` 标记触发 `in_notes = True`
- 注释重编号（阿拉伯数字从 1 开始）
- 标题去重：跳过正文开头重复的标题行

### 4.5 AI 问答

流程：
1. 用户发送消息 → POST `/api/ai-chat`
2. RAG 搜索（top_k=60，hybrid 模式）
3. 构建上下文 → 调用 LLM API
4. 引用重编号 → 扣除使用次数 → 返回答案

余额系统：管理员/终身用户 ∞，普通用户按次（`User.balance`）

### 4.6 搜索系统

三种模式：`semantic_only` / `keyword_only` / `hybrid`
- 嵌入模型 ONNX 768 维，存储在 `/opt/hongloumeng/embed_model/`
- Chroma 向量数据库
- 关键词搜索使用倒排索引

### 4.7 会话管理

- API：`/api/chat/sessions`（CRUD）
- 前端：边栏展示会话列表，`localStorage` 保存当前会话 ID
- 存储：SQLite（Flask 默认）

## 五、已知问题（待修复）

按优先级排列：

### P0 — 图书下载 404

`library.html` 中每个图书卡片 `href="/books/{{ b.path }}"`，但 Flask 没有对应路由。需要添加：

```python
@app.route("/books/<path:filename>")
```

### P0 — 首页布局异常

1. **横向溢出**：`.main` 的 `max-width: 1200px + padding-left: 268px` 导致总宽度 ~1492px，已加 `overflow-x: hidden` 缓解但非根治
2. **AI 问答按钮不响应**：待查。检查是否有 JS 错误阻止事件绑定，或 z-index/偏移导致按钮不可点击
3. **底部名言区消失**：`#quote-bar` 在 `.main-content` 的 flex 布局中，检查 `min-height: calc(100vh - 52px)` 配合 `margin-top: auto` 是否符合预期
4. **纵向滚动条可能不出现**：怀疑 `overflow-x: hidden` 在某些浏览器影响 `overflow-y`

修复方向：
- 改用 `box-sizing: border-box` 使 `max-width` 包含 padding
- 或者只在首页用 CSS 变量或其他方式调整 `max-width`
- `.index-sidebar` 折叠/展开时需确认 `.main` 的 padding 同步正确

### P1 — 搜索索引状态页

`/status` 路由读取 Chroma 索引信息，需要确认索引是否完整构建。

### P2 — 原著注释检测

汉字注释 `〔一〕` 开头可能没有正确触发 `in_notes = True`，影响第 6、7、8、14 等约 30 回的注释分离。

### P2 — 浏览器缓存

JS/CSS 通过 `?v=2` 版本号缓存破坏。每次修改 JS/CSS 后递增。

### P2 — style.css 遗留杂乱

多次直接 sed 修改导致某些规则写在一行、重复声明。不影响功能但影响可维护性。

### P2 — 搜索 API 超时

当前搜索超时设为 150 秒，AI 问答超时 300 秒。如果搜索语料较大，Chroma 查询可能超时。
