# 边栏移至最左 + 推拉拖拽 — 改动记录

## 目标

1. 边栏固定在视口最左侧（脱离 1200px `.main` 容器）
2. 拖拽边栏边缘可调整宽度（类似 VS Code 推拉式）
3. 仅首页（index.html）显示，不影响其他页面

## 改动文件

### `app/templates/index.html`

- 在 `{% block content %}` 内、`.main-content` 前新增 `<aside class="index-sidebar">`：
  - 标题 + 新建对话按钮 + 折叠按钮
  - 会话列表区域（`#sidebar-sessions`）
- `.sidebar-handle` 作为 `<aside>` 的兄弟元素（CSS 相邻选择器隐藏手柄）
- `<style>` 块新增：
  - `body { overflow-x: hidden; }` — 防止横向滚动
  - `.main` — `padding-left: 268px; max-width: calc(1200px + 268px + 1.5rem); box-sizing: border-box;`
  - `.main.sidebar-collapsed` — `padding-left: 0`
  - 移动端 `@media (max-width: 768px)` — `.main { padding-left: 0 }`

### `app/static/style.css` — 末尾新增 ~100 行

```css
.index-sidebar {
    position: fixed;
    top: 52px;
    left: 0;
    width: 250px;
    height: calc(100vh - 52px);
    background: var(--xuanzhi-light);
    border-right: 1px solid var(--border);
    z-index: 90;
    display: flex;
    flex-direction: column;
    transition: transform 0.3s ease;
    overflow: hidden;
    box-sizing: border-box;
}
```

完整规则：`.index-sidebar` / `.collapsed` / `.sidebar-header` / `.sidebar-sessions` / `.sidebar-session-item` / `.sidebar-handle` / `.collapsed + .sidebar-handle` + 移动端 hide。

### `app/static/script.js`

**全局函数重写**（原为 `/* sidebar removed */` 空函数）：

| 函数 | 功能 |
|------|------|
| `loadSessions()` | GET `/api/chat/sessions`，渲染会话列��� |
| `newChatSession()` | POST 新会话，重置聊天界面 |
| `switchSession(sid)` | 切换会话，加载历史消息 |
| `deleteSession(sid)` | 删除会话 |
| `toggleSidebar()` | 折叠/展开边栏，同步 `.main` 的 `padding-left` |
| `startRename(sid, title)` | 行内重命名 |
| `_addSessionToSidebar(title)` | 快捷调用 `loadSessions()` |

**拖拽逻辑**（自执行 IIFE）：

- `mousedown` / `mousemove` / `mouseup` 三件套
- 宽度 clamp `[180, 500]`
- 同时更新 sidebar 的 `width`、handle 的 `left`、`.main` 的 `padding-left`
- `mouseup` 时保存 `hlm_sidebar_width` 到 `localStorage`

**初始化状态恢复**：

- 页面加载时从 `hlm_sidebar_collapsed` 恢复折叠状态
- 从 `hlm_sidebar_width` 恢复宽度和 `.main` 的 `padding-left`

## 已知问题（待下轮调整）

### P0 — `.main` 溢出问题（已部分修复）

`max-width: 1200px` + `padding-left: 268px` 导致 `.main` 总宽 ~1492px，窄屏时水平溢出。已加 `body { overflow-x: hidden }` + `max-width: calc(1200px + 268px + 1.5rem)` 缓解，但效果待验证。

### P0 — AI 问答按钮疑似不响应

- 原因待查。JS 侧的 `addEventListener` 绑定正常，可能为 CSS 偏移导致按钮视觉/交互异常
- 检查方向：`.search-form` / `.search-btn / .ai-btn` 是否被 padding 推偏或 z-index 被 sidebar 遮盖

### P0 — 底部滚动条/名言区消失

- 怀疑与 `.main-content` 的 `min-height: calc(100vh - 52px)` 和 `margin-top: auto` 的交互有关
- `.site-credit` 在 `quote-bar` 之后，可能被推到视口外

### P1 — 边栏宽度/位置微调

- 当前默认宽度 250px，`.main` padding 268px（含 18px 余量），余量偏大
- 建议调整为 `256px`（250+6）或统一用 JS 计算
