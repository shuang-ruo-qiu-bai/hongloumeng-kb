# 边栏移至最左 + 推拉拖拽

## 背景

当前边栏在 `.main` 容器内（`max-width: 1200px; margin: 0 auto;`），无法触及视口最左边缘。用户希望：
1. 边栏固定在视口最左侧（不在 1200px 容器内）
2. 拖拽边栏边缘可调整宽度（像 VS Code 的推拉式），而非仅点按切换

## 方案

### 布局：`position: fixed`

边栏改为 `position: fixed`（桌面端也固定），脱离文档流，相对视口定位：
- 左侧紧贴视口左边缘
- 上对齐到 nav 下方（`top: 52px` 左右）
- 全高度

侧边栏的 handle 也固定定位，位于边栏右边缘。

主内容区域（`.main`）用 `padding-left` 补偿，确保内容不被固定边栏挡住。

### 关键问题：`.main` 是全局的

`.main` 是 base.html 中的公共容器，所有页面共用。需要仅在 index.html 页面上应用左 padding。

**方案**：在 index.html 的 `{% block head %}` 中注入额外 CSS：

```html
<style>
.main { padding-left: 268px; }
</style>
```

拖拽改变宽度时，通过 JS 修改这个 padding-left 值。

### 拖拽逻辑

- handle 上监听 `mousedown`
- `document` 上监听 `mousemove` / `mouseup`
- 拖拽时：更新边栏 `width` 和 `.main` 的 `padding-left`
- 松开时：保存宽度到 `localStorage`（`hlm_sidebar_width`）
- 页面加载时从 `localStorage` 恢复宽度

## 改动文件

| 文件 | 改动 |
|------|------|
| `app/static/style.css` | 边栏 `position: fixed`；handle 固定定位；移除移动端专用 fixed（桌面端通用）；`.main` 边栏覆盖样式 |
| `app/templates/index.html` | `{% block head %}` 中添加边栏启用样式（`.main` 左 padding） |
| `app/static/script.js` | 拖拽事件处理；localStorage 存储宽度 |

## 无需改动

- `app/templates/base.html` — 不变
- `app/app.py` — 不变
- 其他页面模板 — 不受影响

## 验证

1. 页面加载：边栏紧贴视口最左，nav 之上无覆盖
2. 拖拽 handle：边栏宽度实时变化，主内容区同步调整
3. 刷新页面：宽度从 localStorage 恢复
4. 切换页面（其他路由）：边栏不出现（仅 index.html 生效）
5. 折叠/展开功能依然正常（`toggleSidebar()`）
