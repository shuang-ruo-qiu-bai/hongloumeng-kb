# 原著注释区检测遗漏问题

## 问题描述

原著正文/注释分离的逻辑在 `app.py` 的 `api_canon_chapter()` 中：

```python
if not in_notes and re.match(r"^\[\d+\]", stripped):
    in_notes = True
```

只识别**阿拉伯数字** `[1]` 开头的注释行。但很多章节（如第6、7、8、14、17、26、28回等）的注释区以**汉字数字** `〔一〕` 开头，导致：
1. 汉字注释行混入正文
2. 该行之后的 `[2]` 才开始触发 `in_notes`，导致 `[1]` 的注释内容丢失
3. 序号整体偏移（阿拉伯注释从2开始）

已尝试修复（改为同时检测阿拉伯和汉字标记），但当时被用户中断。尚未验证修复效果。

## 涉及章节

约30+章节的注释区以汉字注释开头，包括：6、7、8、14、17、18、19、26、28、29、41、43、46、50、59、60、62、64、67、68、70、72、76、80、85、86、92、97、103、108、114、118、120回。

## 修复要点

- `app.py` 中检测条件改为 `r"^(?:\[\d+\]|[〔\[]([一二三四五六七八九十百零〇]+)[〕\]])"`
- 验证修改后各章节注释区是否正确分离
- 确认正文中不再有汉字注释标记混入

**How to apply:** 下次继续修复时，验证 app.py 中正则修改是否生效。
