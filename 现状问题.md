## P0 — 必须修复

### 1. 图书下载 404

- **位置**：`app/app.py`（缺少路由），`app/templates/library.html`（链接目标）
- **表现**：点击图书卡片跳转到 `/books/xxx.epub`，返回 404 Not Found
- **原因**：`library.html` 链接到 `/books/{{ b.path }}`，但 Flask 没有该路由
- **修复**：在 `app.py` 中添加：

```python
@app.route("/books/<path:filename>")
def download_book(filename):
    return send_from_directory("/opt/hongloumeng/books/红楼梦", filename, as_attachment=True)
```

### 2. 原著注释编号跳柱（不连续）

- **文件**：`app/app.py` — `api_canon_chapter` 函数
- **表现**：打开原著阅读，正文中的 `<sup>[1]</sup>` 和底部注释列表的 `[1]` 对不上。比如正文第一个注标的是 `[5]` 但底部注释从 `[1]` 开始列
- **根因**：API 把阿拉伯数字注释从 **1 开始重新编号**（`aidx = 1; aidx++`），但**正文里的原始编号没有被替换**。正文还是原文件的 `[5][6][7]...`，注释列表变成 `[1][2][3]...`
- **具体代码**（app.py 中注释重编号部分）：

```python
arabic_notes = []
aidx = 1
for line in notes_lines:
    m = re.match(r"^\[(\d+)\](.+)", stripped)
    if m:
        arabic_notes.append({"num": aidx, "text": m.group(2).strip()})
        aidx += 1
```

- **修复方向**：
  - 方案A：正文输出时也替换编号为重新编号后的值（body 中的 `[N]` 按顺序重写）
  - 方案B：注释列表保留原始编号，不做重新编号（`num = int(m.group(1))`）
  - 方案C：前端收到注释列表后做映射，在 body 中替换 sup 的编号

### 3. AI 问答按钮不响应

- **文件**：`app/static/script.js` + `app/static/style.css`
- **表现**：点击 AI 问答按钮无反应（或看似无反应）
- **可能原因**：
  - JS 加载/DOMContentLoaded 正常，事件绑定正常，但 CSS 偏移导致按钮不可视或点击区域偏移
  - 排查 `#ai-btn` 的 z-index、位置、点击事件冒泡
  - 检查浏览器 console 是否有 JS 错误

### 4. 底部名言区消失 / 纵向滚动条异常

- **表现**：首页底部 `#quote-bar` 名言区不可见，纵向滚动条可能不出现
- **根因**（推测）：
  - `body { overflow-x: hidden }` 在某些浏览器（如 Chrome/Safari）上会影响 `overflow-y` 计算
  - `.main-content` 的 flex 布局 `min-height: calc(100vh - 52px)` + `margin-top: auto` 组合有缺陷
  - 修复方向：改用 `html { overflow-x: hidden }`，或去掉 overflow-x 而用其他方式防溢出
